## 1. 토론 주제 및 합의 사항

### 1.1 앱 분리 전략

**PM 의견:** 3개 URL 경로로 분리 (/admin, /referee, /spectator)
**아키텍트 의견:** Option A(단일 앱 + URL 경로) + React.lazy 코드 분할

**합의:**
- 단일 앱 + URL 경로 모드 전환 (Option A) 채택
- React.lazy로 모드별 코드 분할하여 관람자는 관리 코드 다운로드 불필요
- QR 코드로 `/spectator/tournament-id` 직접 진입 가능
- 근거: 개발 효율 + 체육관 현장에서 URL 공유만으로 접근 가능

### 1.2 인증 체계

**PM 의견:** PIN 기반 인증 (관리자 4~6자리, 심판 4자리), 5회 실패 시 30초 잠금
**아키텍트 의견:** Firebase Auth 없이 클라이언트 PIN 검증, SHA-256 해시 저장

**합의:**
- Firebase Auth 미사용 (시각장애인 접근성 + 현장 즉시 사용)
- 관리자 PIN: 대회 단위 설정, sessionStorage 저장
- 심판 PIN: 개인별 4자리, sessionStorage 저장
- 관람자: 인증 없음, localStorage에 UUID(즐겨찾기용)
- PIN은 SHA-256 해시로 Firebase에 저장

### 1.3 데이터 모델

**PM 의견:** Tournament 상위 개념 도입, 대회별로 선수/경기/스케줄 관리
**아키텍트 의견:** Tournament 중심 구조 + 비정규화(관람 모드 경량 구독)

**합의 - 최종 데이터 구조:**

```
/config/
  adminPin: "hashed"              # 앱 전체 관리자 PIN

/tournaments/{tournamentId}/
  name, date, type, status, gameConfig, teamMatchSettings, createdAt

/players/{playerId}/
  name, club, class, createdAt

/referees/{refereeId}/
  name, role, pin(해시), createdAt

/courts/{courtId}/
  name, location, assignedReferees[], createdAt

/tournamentPlayers/{tournamentId}/
  {playerId}: true                # 대회별 참가 선수

/tournamentReferees/{tournamentId}/
  {refereeId}: { assignedMatchIds: [...] }

/matches/{tournamentId}/{matchId}/
  type: "individual" | "team"
  status, courtId, refereeId, scheduledTime, round
  player1Id, player2Id, player1Name, player2Name  # 비정규화
  courtName, refereeName                           # 비정규화
  winnerId, sets[], currentSet
  player1Timeouts, player2Timeouts, activeTimeout
  # 팀전인 경우
  team1, team2, individualMatches[]
  createdAt, updatedAt

/teams/{tournamentId}/{teamId}/
  name, memberIds[], memberNames[]

/schedule/{tournamentId}/{slotId}/
  matchId, courtId, scheduledTime, team1Name, team2Name, status

/favorites/{deviceId}/
  {playerId}: true

/notifications/{tournamentId}/{notifId}/
  type, matchId, playerIds[], message, timestamp
```

**비정규화 이유:** 관람 모드에서 matches만 구독하면 선수명/경기장명 바로 표시 가능. players 전체 구독 불필요.

### 1.4 대회 유형 및 진행 방식

**합의:**

| 대회 유형 | 진행 방식 | 순위 기준 |
|-----------|----------|----------|
| 개인전 | 풀리그 (라운드로빈) | 승수 > 세트 득실 > 포인트 득실 |
| 개인전 | 토너먼트 (싱글 엘리미네이션) | 최종 라운드 기준 |
| 팀전 | 풀리그 | 팀 승수 > 개인전 승수 > 포인트 득실 |
| 랜덤 팀리그전 | 풀리그 | 기존 방식 유지 (팀 승수 기반) |

### 1.5 알림 시스템

**PM 의견:** Web Push + 인앱 토스트 + 음성 안내(TTS) + 진동
**아키텍트 의견:** Firebase onValue 기반 인앱 알림 (Web Push보다 안정적)

**합의:**
- 1차: Firebase onValue 기반 인앱 알림 (토스트 + 진동)
- 2차(추후): Web Push, TTS 음성 안내
- 경기 시작/종료 시 심판 앱이 notifications 노드에 알림 자동 생성
- 관람 앱이 notifications 구독 → 즐겨찾기 선수 필터 → 토스트 표시

### 1.6 접근성

**합의:**
- 관람 모드 최우선 (시각장애인 관람자 다수)
- aria-live="assertive" 점수 변경 안내
- 스크린리더 최적화 (경기 상태 전체 맥락 읽기)
- 고대비 유지 (WCAG AAA 7:1)
- 글씨 크기 3단계 조절 (보통/크게/매우 크게)
- 최소 터치 타겟 48px 유지

## 2. 대회 운영 전체 플로우 (합의안)

### 관리자 플로우
```
1. 관리자 모드 진입 (PIN 인증)
2. "새 대회 만들기" 선택
3. 대회 유형 선택 (개인전/팀전/랜덤 팀리그전)
4. 대회 기본 정보 입력 (이름, 날짜)
5. 게임 설정 (승점 11/21/31, 세트 1/2/3, 진행방식)
6. 참가 선수 등록 (기존 선수 선택 + 신규 등록)
7. 심판 등록 및 PIN 설정
8. 경기장 등록 및 심판 배정
9. 대진표 생성 (자동/수동)
10. 스케줄 설정 (시작 시간, 경기 간격, 경기장 배정)
11. 심판별 담당 경기 배정 (권한 설정)
12. 대회 시작 (status: in_progress)
13. 실시간 대시보드로 전체 경기 모니터링
14. 필요 시 긴급 점수 수정 / 일시정지
15. 모든 경기 완료 확인
16. 순위 확인 및 결과 공유
17. 대회 완료 처리 (status: completed)
```

### 심판 플로우
```
1. 심판 모드 진입
2. 이름 선택 + PIN 입력
3. 배정된 경기 목록 확인 (시간순)
4. 다음 경기 선택
5. 양 선수 정보 확인
6. "경기 시작" 터치
7. 세트별 점수 기록 (+1/-1 큰 버튼)
8. 평터/반칙/타임아웃 기록
9. 세트 종료 → 자동 전환
10. 경기 종료 → 결과 확인
11. 다음 경기로 이동
```

### 관람자 플로우
```
1. 관람 모드 진입 (인증 없음)
2. 진행중 대회 목록에서 대회 선택
3. "실시간 경기" 탭 → 현재 진행중 경기 확인
4. 특정 경기 터치 → 상세 점수 실시간 확인
5. 관심 선수 즐겨찾기 추가 (별 아이콘)
6. "즐겨찾기" 탭 → 관심 선수 경기만 모아보기
7. 즐겨찾기 선수 경기 시작 시 알림 수신
8. "대진표" 탭 → 전체 대진 현황
9. "순위" 탭 → 현재 순위 확인
10. "히스토리" 탭 → 완료된 경기 결과 조회
```

## 3. 화면 구성 (합의안)

### 3.1 공통
| 화면 | 경로 | 설명 |
|------|------|------|
| 모드 선택 | `/` | 관리/심판/관람 3개 모드 선택 |

### 3.2 관리자 모드
| 화면 | 경로 | 설명 |
|------|------|------|
| 대시보드 | `/admin` | 대회 목록, 진행 상황 요약 |
| 대회 생성 | `/admin/tournament/new` | 대회 유형/설정 입력 |
| 대회 상세 | `/admin/tournament/:id` | 대회 관리 (탭: 선수/대진표/스케줄/권한/순위) |
| 선수 관리 | `/admin/players` | 선수 CRUD |
| 심판 관리 | `/admin/referees` | 심판 CRUD + PIN 설정 |
| 경기장 관리 | `/admin/courts` | 경기장 CRUD + 심판 배정 |
| 실시간 현황 | `/admin/tournament/:id/live` | 모든 경기 실시간 모니터링 |

### 3.3 심판 모드
| 화면 | 경로 | 설명 |
|------|------|------|
| 심판 인증 | `/referee` | 이름 선택 + PIN 입력 |
| 내 경기 목록 | `/referee/games` | 배정된 경기 (시간순) |
| 개인전 점수 기록 | `/referee/match/:tId/:mId` | 전체화면 점수판 |
| 팀전 점수 기록 | `/referee/team/:tId/:mId` | 개별 경기 점수 입력 |

### 3.4 관람 모드
| 화면 | 경로 | 설명 |
|------|------|------|
| 대회 목록 | `/spectator` | 진행중/완료 대회 목록 |
| 대회 관람 | `/spectator/tournament/:id` | 탭: 실시간/대진표/순위/히스토리 |
| 경기 상세 | `/spectator/match/:tId/:mId` | 실시간 점수 (읽기 전용) |
| 즐겨찾기 | `/spectator/favorites` | 즐겨찾기 선수 경기 모아보기 |
