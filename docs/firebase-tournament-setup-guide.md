# Firebase 대형 대회 운영 설정 가이드

## 개요

쇼다운 대회 관리 앱을 대형 대회(10코트+, 50경기+, 200명+ 동시 접속)에서 안정적으로 운영하기 위한 Firebase 설정 가이드입니다.

---

## Step 1. Blaze 플랜 업그레이드 (필수)

### 왜 필요한가?

현재 Spark(무료) 플랜은 **동시 연결 100개**로 제한됩니다. 관람자 50명 + 심판 20명 + 관리자만 해도 100개를 초과합니다.

### 설정 방법

1. [Firebase Console](https://console.firebase.google.com) 접속
2. 프로젝트 선택 (`showdown-b5cc7`)
3. 좌측 하단 **"Spark"** 클릭
4. **"Blaze 플랜으로 업그레이드"** 선택
5. 결제 계정 연결 (신용카드/체크카드)
6. 완료

### 요금 비교

| 항목 | Spark (무료) | Blaze (종량제) |
|------|-------------|----------------|
| 동시 연결 | 100 | 200,000 |
| DB 저장 | 1GB | 무제한 |
| 다운로드 | 10GB/월 | 종량 ($1/GB) |
| 인증 | 무제한 | 무제한 |
| 호스팅 | 10GB/월 | 종량 |
| **예상 비용** | **$0** | **대회 1일 $1~5** |

> **참고:** Blaze 플랜에도 Spark과 동일한 무료 한도가 포함되어 있습니다. 무료 한도를 초과한 사용량에 대해서만 과금됩니다. 소규모 테스트는 $0입니다.

---

## Step 2. 예산 알림 설정 (필수)

과금 폭탄을 방지하기 위해 예산 알림을 설정합니다.

### 설정 방법

1. Firebase Console → 좌측 메뉴 **"사용량 및 결제"** (Usage and billing)
2. **"예산 알림"** (Budget alerts) 클릭
3. **"예산 만들기"** 클릭
4. 예산 금액: **$10** (또는 원하는 금액)
5. 알림 기준 설정:
   - 50% ($5) → 이메일 경고
   - 90% ($9) → 이메일 경고
   - 100% ($10) → 이메일 경고
6. 이메일 알림 활성화
7. **"저장"**

> **중요:** 예산 알림은 과금을 **차단하지 않습니다**. 알림만 보냅니다. 실제 과금 한도를 설정하려면 Google Cloud Console에서 결제 계정의 예산 + 자동 중지를 설정하세요.

---

## Step 3. App Check 활성화 (권장)

앱 외부에서 Firebase API를 직접 호출하는 것을 차단합니다. 악의적 사용자가 스크립트로 데이터를 조작하는 것을 방지합니다.

### 설정 방법

1. [reCAPTCHA Admin](https://www.google.com/recaptcha/admin) 접속
2. **"+"** 클릭하여 새 사이트 등록
   - 라벨: `showdown-tournament`
   - reCAPTCHA 유형: **reCAPTCHA v3**
   - 도메인: `showdown-b5cc7.web.app`
   - 약관 동의 → **제출**
3. **사이트 키**와 **비밀 키**를 복사

4. Firebase Console → 좌측 메뉴 **"App Check"**
5. **"시작하기"** 클릭
6. 웹앱 선택
7. 공급자: **"reCAPTCHA v3"** 선택
8. 사이트 키 붙여넣기
9. **"저장"**

10. **"Realtime Database"** 탭 클릭
11. **"적용"** (Enforce) 클릭

> **비용:** 무료

---

## Step 4. Realtime Database 규칙 확인

앱에서 이미 보안 규칙을 배포했습니다. 올바르게 적용되었는지 확인합니다.

### 확인 방법

1. Firebase Console → **"Realtime Database"**
2. **"규칙"** 탭 클릭
3. 다음 규칙이 포함되어 있는지 확인:
   - `matches` 노드에 `updatedAt` 검증 규칙
   - `$other: { ".validate": false }` 필드 제한
   - `.read: true` (관람자 접근 허용)

### 규칙 재배포 (필요 시)

```bash
npx firebase deploy --only database
```

## Step 5. Performance Monitoring 활성화 (선택)

대회 중 앱 성능을 실시간으로 모니터링합니다.

### 설정 방법

1. Firebase Console → 좌측 메뉴 **"Performance"**
2. **"시작하기"** 클릭
3. 자동으로 웹앱 성능 데이터 수집 시작

### 모니터링 항목

- 페이지 로딩 시간
- 네트워크 요청 지연
- Firebase 읽기/쓰기 응답 시간

> **비용:** 무료

---

## Step 6. 대회 전 데이터 정리 (권장)

불필요한 테스트 데이터가 많으면 동기화 속도가 느려질 수 있습니다.

### 정리 방법

1. Firebase Console → **"Realtime Database"** → **"데이터"** 탭
2. 오래된 테스트 대회 데이터 확인
3. 불필요한 데이터 삭제 (앱 관리자 모드에서 대회 삭제 가능)

> **주의:** 실제 대회 데이터를 삭제하지 않도록 주의하세요.

---

## Step 7. 자동 백업 설정 (선택, Blaze만 가능)

대회 데이터 분실을 방지하기 위해 자동 백업을 설정합니다.

### 설정 방법

1. Firebase Console → **"Realtime Database"**
2. **"백업"** 탭 클릭
3. **"자동 백업 설정"**
4. Google Cloud Storage 버킷 선택 (자동 생성됨)
5. 백업 주기: **매일**
6. **"저장"**

### 수동 백업

대회 직전에 수동 백업을 권장합니다:

1. **"백업"** 탭 → **"지금 백업"** 클릭

> **비용:** Cloud Storage 저장 비용 소액 발생 (대회 데이터 기준 ~$0.01)

## 대회 당일 체크리스트

### 사전 준비 (대회 1일 전)

```
☐ Blaze 플랜 활성화 확인
☐ 예산 알림 설정 확인
☐ 대회 데이터 입력 완료 (선수, 심판, 경기장)
☐ 대진표 생성 완료
☐ 스케줄 배정 완료
☐ 심판별 경기 배정 확인
☐ 수동 백업 1회 실행
```

### 현장 테스트 (대회 당일 아침)

```
☐ 대회장 WiFi 연결 테스트 (10+ 기기 동시 접속)
☐ 관리자 PIN으로 로그인 확인
☐ 심판 1명 로그인 + 테스트 경기 점수 기록
☐ 관람 모드에서 실시간 점수 확인
☐ PDF 기록지 다운로드 테스트
☐ 모바일 데이터 백업 경로 확인 (WiFi 장애 대비)
```

### 대회 진행 중

```
☐ Firebase Console에서 동시 연결 수 모니터링
☐ 관리자는 진행 중 경기가 있을 때 대진표 수정 금지
☐ 심판은 브라우저 탭 닫지 않기 (세션 유지)
☐ WiFi 불안정 시 모바일 데이터로 전환 안내
☐ 각 코트별 경기 완료 후 다음 경기 즉시 시작
```

### 대회 종료 후

```
☐ 전체 경기 결과 확인 (관리자 → 순위 탭)
☐ 순위표 CSV 내보내기
☐ 개별 경기 PDF 기록지 다운로드 (필요 시)
☐ Firebase Console에서 사용량/비용 확인
☐ 대회 데이터 백업 (수동 백업 또는 자동 백업 확인)
```

## 동시 접속 규모별 권장 설정

| 규모 | 동시 접속 | 플랜 | 예상 비용/일 | 추가 설정 |
|------|----------|------|-------------|----------|
| 소규모 | ~50명 | Spark | $0 | 없음 |
| 중규모 | 50~200명 | Blaze | $1~3 | 예산 알림 |
| 대규모 | 200~500명 | Blaze | $3~10 | App Check + 예산 알림 |
| 초대규모 | 500명+ | Blaze | $10~30 | App Check + 성능 모니터링 + 백업 |

## 문제 발생 시 대응

### "연결이 끊어졌습니다" 메시지

1. WiFi 상태 확인
2. 모바일 데이터로 전환
3. 앱 새로고침 (F5 또는 당겨서 새로고침)

### 심판 앱이 멈춤

1. 브라우저 탭 닫지 말 것
2. 새로고침 → 자동으로 진행 중인 경기 복구 안내 표시
3. "경기 이어하기" 버튼 클릭

### 점수가 반영되지 않음

1. 화면 하단 오프라인 배너 확인
2. 오프라인 상태라면 점수는 자동 저장됨
3. 네트워크 복구 시 자동 동기화

### 동시 접속 제한 도달

1. Firebase Console → Realtime Database → 사용량 확인
2. Blaze 플랜이 아닌 경우 즉시 업그레이드
3. 관람자에게 새로고침 자제 요청
