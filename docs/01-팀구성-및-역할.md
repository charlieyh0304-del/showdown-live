# 에이전트 팀 구성 및 역할 분담

## 프로젝트 개요
쇼다운(시각장애인 테이블 스포츠) 대회 관리 앱을 관람/심판/관리자 3개 모드로 분리 재설계

## 팀 구성 (6명)

| # | 역할 | 담당 업무 |
|---|------|----------|
| 1 | **PM (기획자)** | 요구사항 정의, 사용자 플로우, 화면 설계, 접근성 기획 |
| 2 | **아키텍트 (설계자)** | 기술 아키텍처, 데이터 모델, Firebase 구조, 인증 체계 |
| 3 | **관리자 모드 개발자** | 대회 생성/관리, 선수/심판/경기장 CRUD, 권한 설정, 순위 산출 |
| 4 | **심판 모드 개발자** | 심판 인증, 배정 경기 표시, 개인전/팀전 점수 기록 |
| 5 | **관람 모드 개발자** | 실시간 관람, 즐겨찾기, 알림, 접근성 최적화 |
| 6 | **QA (검증자)** | 빌드 검증, 타입 체크, 통합 테스트, 접근성 검사 |

## 작업 단계

### Phase 1: 기획 + 설계 (PM + 아키텍트)
- 요구사항 정의 → 기술 아키텍처 설계 → 교차 검토 → 합의

### Phase 2: 공통 기반 구축 (아키텍트)
- 타입 정의, 데이터 모델, 공통 유틸리티, 프로젝트 구조 리팩토링

### Phase 3: 모드별 병렬 개발 (개발자 3명)
- 관리자/심판/관람 모드 동시 개발

### Phase 4: 통합 검증 (QA)
- 빌드, 타입 체크, 모드 간 데이터 연동 검증
